# Introduction

GitHub Copilot **Agent Mode** goes beyond code completion. It can understand a high-level objective, create a plan, modify multiple files, generate tests, fix compilation errors, and iterate until the task is complete.

In this lab you will use **Agent Mode** to implement a complete feature in the **eShopOnWebITC** repository.

---

# Learning Objectives

After completing this lab you will be able to:

- Enable GitHub Copilot Agent Mode in VS Code
- Assign a multi-step development task
- Review the execution plan generated by the agent
- Observe multi-file changes
- Review generated tests
- Validate the implementation before committing

# Enterprise Scenario

The Product team has requested a **Wishlist** feature.

Requirements:

- Customers can add products to a wishlist.
- Customers can remove products.
- Display wishlist count.
- Persist wishlist between sessions.
- Create unit tests.

Instead of implementing each task manually, you will delegate the work to GitHub Copilot Agent Mode.

---

# Agentic Workflow

```text
Feature Request
      │
      ▼
Copilot Agent
      │
      ├── Analyze Repository
      ├── Create Plan
      ├── Modify Multiple Files
      ├── Generate Tests
      ├── Fix Errors
      └── Present Summary
      │
      ▼
Developer Review
```

# Exercise 1 – Enable Agent Mode

1. Open **eShopOnWebITC** in VS Code.
2. Sign in to GitHub.
3. Verify GitHub Copilot is active.
4. Open GitHub Copilot Chat.
5. Switch the chat experience to **Agent Mode** (if available in your environment).

Verify the agent can inspect your workspace.

---

# Exercise 2 – Ask the Agent to Build the Feature

Provide a goal similar to:

> Build a Wishlist feature for the eShopOnWebITC application. Reuse existing repository patterns, generate unit tests, update documentation, and fix any compilation errors.

Observe whether the agent:

- Creates a development plan
- Identifies impacted files
- Explains assumptions
- Requests clarification when needed

---

# Exercise 3 – Review the Plan

Before accepting changes, inspect:

- Planned tasks
- Files to modify
- Dependencies
- Test strategy
- Potential risks

Discuss:

- Would you change the implementation approach?
- Is any business requirement missing?

---

# Exercise 4 – Observe Multi-File Changes

Review changes made across:

- Controllers
- Services
- Models
- Views/UI
- Dependency Injection
- Tests

Check that repository coding standards are followed.

# Exercise 5 – Validate the Feature

Build the application.

Run:

```bash

dotnet test
```

Verify:

- Build succeeds

- Wishlist works as expected

# Exercise 6 – Human Review

Before committing:

- Review every file
- Reject unnecessary changes
- Improve prompts if needed
- Ask the agent to refine specific files

Remember: Agent Mode accelerates development but **developer approval is still required**.

# Best Practices

- Start with a clear objective.
- Let the agent create a plan before coding.
- Review every generated change.
- Run builds and tests after execution.
- Keep a human in the loop for production code.
