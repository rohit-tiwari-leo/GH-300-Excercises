# Lab 4: Build a RAGAS Evaluation Harness to Measure Faithfulness in a RAG System

## Objective

In this lab, you will build a simple evaluation harness using **RAGAS** to measure the **faithfulness** of responses generated by a Retrieval-Augmented Generation (RAG) system. You will create a small evaluation dataset, run the evaluation, and interpret the faithfulness score.

**Estimated Duration:** 30 minutes

By the end of this lab, you will be able to:

- Understand the concept of faithfulness in RAG systems

# Lab Scenario

Your organization has developed a RAG-based internal knowledge assistant that answers employee questions using company documentation.

Before deploying the assistant, you want to verify that the generated answers are **faithful to the retrieved documents** and do not introduce unsupported or hallucinated information.

---

# Step 1: Create the Project

Create the following project structure.

```text
ragas-faithfulness/
│
├── evaluate.py
└── dataset.py
```

---

# Step 2: Create a Sample Evaluation Dataset

Open **dataset.py** and create a small evaluation dataset.

```python
from datasets import Dataset

dataset = Dataset.from_dict({
    "question": [
        "What is GitHub Copilot?"
    ],
    "contexts": [[
        "GitHub Copilot is an AI-powered coding assistant developed by GitHub in collaboration with OpenAI. It helps developers write code faster by providing code suggestions."
    ]],
    "answer": [
        "GitHub Copilot is an AI-powered coding assistant that helps developers write code faster."
    ]
})
```

The dataset contains:

- User question
- Retrieved document (context)
- Generated answer from the RAG system

---

# Step 3: Create the Evaluation Script

Create **evaluate.py**.

```python
from dataset import dataset
from ragas import evaluate
from ragas.metrics import faithfulness

result = evaluate(
    dataset,
    metrics=[faithfulness]
)

print(result)
```

---

# Step 4: Run the Evaluation

Execute the evaluation.

```bash
python evaluate.py
```

Example output:

```text
Evaluating...

faithfulness

0.98
```

---

# Step 5: Interpret the Faithfulness Score

Suppose the evaluation returns:

```text
Faithfulness Score: 0.98
```

Interpretation:

- The generated answer is almost completely supported by the retrieved context.
- Very little or no hallucination is present.

---

# Example of a Faithful Answer

### Retrieved Context

```text
GitHub Copilot is an AI-powered coding assistant developed by GitHub in collaboration with OpenAI.
```

### Generated Answer

```text
GitHub Copilot is an AI-powered coding assistant developed by GitHub.
```

**Faithfulness Score**

```text
0.99
```

**Result:** PASS

---

# Example of an Unfaithful Answer

### Retrieved Context

```text
GitHub Copilot is an AI-powered coding assistant developed by GitHub.
```

### Generated Answer

```text
GitHub Copilot was created by Microsoft in 2018 and supports more than 100 programming languages.
```

The generated answer introduces information that is **not present in the retrieved context**.

Example output:

```text
Faithfulness Score

0.42
```

**Result:** FAIL

---

# Step 6: Define Evaluation Threshold

Use the following guideline to interpret the results.

| Faithfulness Score | Interpretation |
|--------------------|----------------|
| 0.90 – 1.00 | Excellent |
| 0.75 – 0.89 | Good |
| 0.60 – 0.74 | Needs Improvement |
| Below 0.60 | Poor (Possible Hallucination) |
