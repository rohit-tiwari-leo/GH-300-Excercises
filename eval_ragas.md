# Lab 4: Build a RAGAS Evaluation Harness to Measure Faithfulness in a RAG System

## Objective

In this lab, you will build a simple evaluation harness using **RAGAS** to measure the **Faithfulness** of responses generated by a Retrieval-Augmented Generation (RAG) system. You will create an evaluation dataset, configure an evaluation LLM, run the evaluation, and interpret the resulting faithfulness score.

# Lab Scenario

Your organization has developed an AI-powered knowledge assistant using Retrieval-Augmented Generation (RAG). The assistant retrieves information from an internal knowledge base and uses GitHub Copilot to generate answers.

Before deploying the solution, you want to verify that the generated answers are **faithful to the retrieved documents** and do not introduce unsupported or hallucinated information.


Linux/macOS

```bash

```

> **Note**
>
> If your organization uses GitHub Models or another supported provider, configure the corresponding endpoint and credentials instead.

# Step 3: Create the Evaluation Dataset

Open **dataset.py** and create a simple evaluation dataset.

```python
from datasets import Dataset

dataset = Dataset.from_dict({
    "question": [
        "What is GitHub Copilot?"
    ],
    "contexts": [[
        "GitHub Copilot is an AI-powered coding assistant developed by GitHub in collaboration with OpenAI. It helps developers write code faster by providing intelligent code suggestions."
    ]],
    "answer": [
        "GitHub Copilot is an AI-powered coding assistant that helps developers write code faster."
    ]
})
```

The dataset contains:

- User Question
- Retrieved Context
- Generated Answer

---

# Step 4: Configure the Evaluation Harness

Open **evaluate.py**.

Configure the evaluation LLM.

Example using OpenAI:

```python
from langchain_openai import ChatOpenAI

evaluator_llm = ChatOpenAI(
    model="gpt-4o-mini",
    temperature=0
)
```

Then create the RAGAS evaluation.

```python
from dataset import dataset
from ragas import evaluate
from ragas.metrics import faithfulness

result = evaluate(
    dataset,
    metrics=[faithfulness],
    llm=evaluator_llm
)

print(result)
```

> **Note**
>
> If your organization uses Azure OpenAI or GitHub Models, initialize the appropriate client instead of `ChatOpenAI`.

# Step 6: Interpret the Results

### Example 1 – Faithful Response

### Retrieved Context

```text
GitHub Copilot is an AI-powered coding assistant developed by GitHub in collaboration with OpenAI.
```

### Generated Answer

```text
GitHub Copilot is an AI-powered coding assistant developed by GitHub.
```

Evaluation Output

```text
Faithfulness Score

0.99
```

**Result:** PASS

The generated answer is fully supported by the retrieved context.

---

### Example 2 – Unfaithful Response

### Retrieved Context

```text
GitHub Copilot is an AI-powered coding assistant developed by GitHub.
```

### Generated Answer

```text
GitHub Copilot was released in 2021 and supports over 100 programming languages.
```

Evaluation Output

```text
Faithfulness Score

0.43
```

**Result:** FAIL

The generated answer contains information that is **not supported** by the retrieved context.

# Step 7: Define Evaluation Thresholds

Use the following guideline to interpret the evaluation score.

| Faithfulness Score | Interpretation |
|--------------------|----------------|
| 0.90 – 1.00 | Excellent |
| 0.75 – 0.89 | Good |
| 0.60 – 0.74 | Needs Improvement |
| Below 0.60 | Poor (Possible Hallucination) |

# How the Evaluation Works

```text
User Question
      │
      ▼

      │
      ▼

      │
      ▼
Create Evaluation Dataset

      │
      ▼
RAGAS sends the data to the Evaluation LLM
      │
      ▼
Evaluation LLM determines whether
the answer is grounded in the context
      │
      ▼

```
